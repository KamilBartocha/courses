Konfigurowanie repozytorium Git (integracja z PyCharm)

Instrukcja dodatkowa dla osb chcych wsppracowa z systemem kontroli wersji

Gdy chcesz umieci istniejcy projekt pod kontrol wersji Git, PyCharm automatycznie wykryje, czy Git jest zainstalowany na Twoim komputerze. Jeli IDE nie moe zlokalizowa pliku wykonywalnego Git, sugeruje jego pobranie.

Umieszczanie istniejcego projektu kodu rdowego pod kontrol wersji Git

Moesz utworzy lokalne repozytorium Git na podstawie istniejcych rde projektu.

Powi cay projekt z jednym repozytorium Git

  1. Otwrz projekt, ktry chcesz umieci w Git.
  2. Wybierz opcj Enable Version Control Integration z VCS Operations Popup lub z gwnego menu VCS.
  3. Wybierz Git jako system kontroli wersji i kliknij OK.
  4. Po wczeniu integracji VCS, PyCharm zapyta, czy chcesz udostpnia pliki ustawie projektu za porednictwem VCS. Moesz wybra opcj Always Add, aby zsynchronizowa ustawienia projektu z innymi uytkownikami repozytorium, ktrzy pracuj z PyCharm.

Po zainicjowaniu repozytorium Git dla projektu musisz doda pliki projektu do repozytorium.

Dodaj pliki do lokalnego repozytorium

  1. W oknie narzdzia Commit rozwi wze Unversioned Files.
  2. Wybierz pliki, ktre chcesz doda do Git lub ca list zmian i wybierz Add to VCS z menu kontekstowego.
    Moesz take doda pliki do lokalnego repozytorium Git z okna narzdzia Projekt: wybierz pliki, ktre chcesz doda i wybierz Git | Add z menu kontekstowego.

Dodaj zdalne repozytorium

Jeli utworzye repozytorium Git na podstawie rde lokalnych, musisz doda repozytorium zdalne, aby mc wsppracowa nad projektem Git, a take wyeliminowa ryzyko zwizane z przechowywaniem caej bazy kodu lokalnie. Przesyasz zmiany do zdalnego repozytorium, gdy musisz udostpni swoj prac i pobra z niej dane, aby zintegrowa zmiany wprowadzone przez innych wspautorw z wersj lokalnego repozytorium.

Zdefiniuj zdalne repozytorium

  1. Utwrz puste repozytorium na dowolnym hostingu Git, takim jak Bitbucket lub GitHub.
  2. Wywoaj okno dialogowe Push, gdy jeste gotowa/gotowy do przekazania swoich zatwierdze, wybierajc Git | Push w menu gwnym.
  3. Jeli do tej pory nie dodano adnych zdalnych repozytoriw, zamiast nazwy zdalnego repozytorium pojawi si link Define remote. Kliknij go, aby doda zdalne repozytorium.
  4. W wywietlonym oknie dialogowym okrel nazw zdaln i adres URL, pod ktrym bdzie ona obsugiwana, a nastpnie kliknij przycisk OK.
